How to Update Text in Solara App

⚠️ Before making any changes:
  • Always make a backup of the files you plan to modify
  • Test changes in development environment first
  • Keep the same text formatting (if text includes special characters or formatting)

1. Tutorial Text Updates

To modify the tutorial steps and content:

src/components/TutorialModal.js

Find the tutorialSteps array at the beginning of the file. Each step has a title and content:

const tutorialSteps = [ { title: "Get Started", content: "Welcome! Type your questions or commands..." }, { title: "Manage Chats", content: "Click the '+' button in the sidebar to start a new chat..." }, // ... other steps ];

Modify the title and content properties as needed. Keep the same array structure.

2. Login Page Text Updates

src/pages/LoginPage.js

To update the login page text:

  • Main title: Find <h1> tag with "Welcome to Solara"
  • Subtitle: Find <h2> tag with "Your Pocket Communication Coach"
  • Welcome message: Find <p> tag with "Please sign in to continue"
  • Button text: Look for the button elements containing "Sign in with Google" and "Sign in with Email Link"

3. Chat Interface Text Updates

src/components/ChatInterface.js

To modify chat interface text:

  • Header: Find <h2> tag containing "Solara" in the top navigation bar
  • Welcome Modal: Look for the WelcomeModal component section:
    • Title: "Welcome, {user?.displayName || 'New User'}!"
    • Welcome text: "Thank you for joining our AI assistant platform!"
    • Feature list: Update the <ul> items in the modal
    • Button text: "Get Started"
  • Input placeholder text: Find the textarea element with placeholder props like "Type your message..."
  • Button labels: Look for buttons containing "New Chat", "Account", "Logout", etc.

4. Error Messages and Notifications

Error messages and notifications are spread throughout the components. Common locations:

  • Login errors: In LoginPage.js, look for setError calls
  • Chat errors: In ChatInterface.js, look for setError calls
  • Subscription messages: Search for text in subscription-related components
Important Notes:
  • Some text includes dynamic content using variables (e.g., {user.displayName}). Keep these dynamic elements when updating text.
  • Maintain proper quotation marks and string formatting
  • After making changes, test the application to ensure everything displays correctly
  • If you're not comfortable with code edits, please work with a developer

5. Deployment to Google Cloud Run

Follow these steps to deploy the application to Google Cloud Run:

Step 1: Install Google Cloud CLI

  1. Visit https://cloud.google.com/sdk/docs/install
  2. Download and install the Google Cloud CLI for your operating system
  3. Verify installation by opening a terminal and running: gcloud --version

Step 2: Authentication and Project Setup

  1. Log in to Google Cloud:
    gcloud auth login

    Follow the browser prompts to authenticate your account

  2. Set the project:
    gcloud config set project solara-459523

Step 3: Prepare for Deployment

  1. Navigate to your project directory in the terminal
  2. Open .gitignore and comment out the .env line:
    # dependencies /node_modules /.pnp .pnp.js # testing /coverage # production /build # misc .DS_Store #.env # Comment out this line .env.local .env.development.local .env.test.local .env.production.local
  3. Make sure your .env file contains the correct production values

Step 4: Deploy

  1. Run the deployment command:
    gcloud run deploy gpt-assistant-app --source . --region asia-southeast1

    This will build and deploy your application. The process may take a few minutes.

  2. After successful deployment, Google Cloud will provide you with the service URL where your application is accessible.

Step 5: Post-Deployment

  1. Once deployment is complete, uncomment the .env line in .gitignore:
    # misc .DS_Store .env # Uncomment this line .env.local
  2. Verify that your application is working correctly at the provided URL
  3. Check the Cloud Run console for logs if you encounter any issues
Deployment Notes:
  • Always verify your .env variables before deploying
  • Make sure your Firebase and other service configurations are set up for production
  • Keep your .env file secure and never commit it to version control
  • Monitor your Cloud Run dashboard for any errors or performance issues
